# ParroTalk API Documentation Hub

Welcome to the comprehensive API documentation for the ParroTalk Dictation System. This document provides guides on how to interact with the backend for audio uploads, transcription processing, and real-time status tracking.

---

## 1. General Information

*   **Host:** `http://localhost:8080` (Localhost)
*   **Base Path:** `/api`
*   **Response Format:** JSON (`application/json`) & SSE (`text/event-stream`)

### Key Error Codes
*(ParroTalk utilizes standard HTTP status codes instead of a custom wrapper)*
| HTTP Code | Description |
| :--- | :--- |
| `200 OK` | Request was successful |
| `400 Bad Request` | Invalid input or file types |
| `500 Internal Error` | Processing engine or Cloudinary connection failure |

## 2. API Endpoints

Below are the detailed documentations for the audio processing subsystem:

### 2.1. Audio Upload & Processing

**Upload Audio**
Upload a physical audio or video file to save to Cloudinary and begin the AI transcription job. Features built-in MD5 deduplication to instantly return cached processing jobs if already detected.

*   **URL:** `/api/audio/upload`
*   **Method:** `POST`
*   **Content-Type:** `multipart/form-data`

**Request Parameters:**
| Field | Type | Description |
| :--- | :--- | :--- |
| `file` | File | The audio/video file to process (mp3, wav, mp4, etc.) |

**Success Response (200 OK):**
```json
{
  "jobId": "4f5e8b0a-...",
  "message": "File uploaded successfully and processing started"
}
```

---

### 2.2. Job Management & Status

**List All Jobs**
Retrieve a descending list of all previously uploaded and processed audio jobs.

*   **URL:** `/api/jobs`
*   **Method:** `GET`

**Success Response (200 OK):**
```json
[
  {
    "id": "4f5e8b0a-...",
    "status": "DONE",
    "progress": 100,
    "currentStep": "Completed",
    "fileUrl": "https://res.cloudinary.com/...",
    "createdAt": "2026-04-10T17:40:00"
  }
]
```

**Get Specific Job Status**
Quickly poll the exact metadata status of a single job.

*   **URL:** `/api/jobs/{jobId}`
*   **Method:** `GET`

**Success Response (200 OK):**
```json
{
  "id": "4f5e8b0a-...",
  "status": "PROCESSING",
  "progress": 45,
  "currentStep": "Running Whisper model",
  "fileUrl": "https://res.cloudinary.com/...",
  "createdAt": "2026-04-10T17:40:00"
}
```

---

### 2.3. Real-Time Tracking (SSE)

**Stream Job Status**
Connect to the backend via Server-Sent Events to receive asynchronous progress ticks in real-time. Automatically connects and pipes outputs mapped from the remote Python AI API.

*   **URL:** `/api/jobs/{jobId}/stream`
*   **Method:** `GET`
*   **Accept:** `text/event-stream`

**Event Stream Payload (JSON emitted over time):**
```json
{
  "status": "PROCESSING",
  "progress": 10,
  "step": "Connected to AI Engine"
}
```
*(Updates are pushed dynamically from the server without requiring polling)*

---

### 2.4. Results Retrieval

**Get Transcription Result**
Retrieve the fully mapped transcription payload containing both full sentences and exact word timestamp locations generated by the AI Engine.

*   **URL:** `/api/jobs/{jobId}/result`
*   **Method:** `GET`

**Success Response (200 OK):**
```json
{
  "sentences": [
    {
      "text": "Hello world.",
      "start": 0.0,
      "end": 2.5
    }
  ],
  "words": [
    {
      "text": "Hello",
      "start": 0.0,
      "end": 1.2
    },
    {
      "text": "world.",
      "start": 1.3,
      "end": 2.5
    }
  ]
}
```

## 3. Helpful SDKs & Tooling

*   **Postman** - For manually pushing test audio mapping directly to `/api/audio/upload`
*   **EventSource API** - Web-standard for mapping SSE strings inside NextJS components (Use this mapped to `/api/jobs/{id}/stream`).
